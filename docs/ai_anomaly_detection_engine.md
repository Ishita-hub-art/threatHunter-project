# AI-Based Behavior Anomaly Detection Engine

## Overview
The AI engine evaluates features extracted from logs (like the number of failed logins or unusual IP addresses) and calculates a numerical score indicating how anomalous or "weird" the event is.

## Purpose of the Module
To replace rigid, static security rules with a machine learning model that learns normal user baselines. If a student account normally accesses educational datasets and suddenly starts attempting to download entire server backups at 3:00 AM, the model flags it.

## Inputs
* **Feature Vectors:** Structured numpy arrays or pandas DataFrames generated by the preprocessing layer from the raw JSON logs.

## Processing Steps
1. **Feature Engineering:** Converts raw text strings (e.g., timestamps, usernames, API names) into numerical features.
   * `Time_Sine/Cosine`: representing time-of-day cyclically.
   * `Velocity`: rate of requests per minute.
2. **Unsupervised ML Modeling:** Employs algorithms like `Isolation Forest` (available in Python's `scikit-learn`) to detect multidimensional outliers.
3. **Scoring:** The model outputs an anomaly score ranging from 0 (completely normal) to 100 (highly anomalous).
4. **Explainability:** Basic feature importance evaluation (e.g., using `SHAP`) to point out *why* the score was high (e.g., "The IP address was highly unusual for this user").

## Outputs
* **Anomaly Records:** The original JSON log entry appended with an `anomaly_score` and an `anomaly_status` boolean.

## Example Data Format
**Input (Feature Vector Array):**
```python
# [hour_of_day, login_failures_15m, data_downloaded_gb]
[3.5, 15, 45.2]
```

**Output (JSON Anomaly Result):**
```json
{
  "user": "student_admin",
  "anomaly_score": 92.4,
  "anomaly_status": "SUSPICIOUS",
  "explanation": "High volume data download (45.2 GB) at unusual hour (3.5 AM)"
}
```
